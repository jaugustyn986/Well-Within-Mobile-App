# Instagram Decision Rubric

Use this when choosing the next Well Within Instagram post. The goal is controlled learning, not random content.

## Default Rule

If there is no clear performance winner, choose the current recommended or highest-fit draft in `docs/social/INSTAGRAM_DRAFT_QUEUE.md`.

Always use `docs/social/INSTAGRAM_MEMORY.md` and recent external-account research as pattern input. Do not copy posts, captions, layouts, or artwork from other accounts.

## Queue Hygiene And Repeat Prevention

Before adding or choosing a draft, check `INSTAGRAM_DRAFT_QUEUE.md`, the queue's `Recently Published` section, and `INSTAGRAM_EXPERIMENT_LOG.md`. If a draft repeats a published or queued post's audience, hook shape, target metric, and visual system, merge it, retire it, or explicitly justify why it should remain as a controlled test.

Prefer a short active queue of distinct jobs:

- reach-first identity
- beginner education
- trust or privacy
- community prompt
- product-light utility

Retire or merge drafts that only restate an existing idea with slightly different wording.

## Performance Confidence

### Low Confidence

Use low confidence when:

- reach is too small to compare
- saves, shares, follows, or profile visits are all near zero
- results are missing or younger than the intended checkpoint
- only one post has tested the concept or format

Action: choose from the draft queue based on strategic fit. Prefer clearer first-frame language, stronger keywords, and one primary metric.

### Medium Confidence

Use medium confidence when:

- one post clearly beats a similar baseline on the target metric
- the result matches the creative hypothesis
- there is enough reach to make the rate meaningful

Action: create a controlled sibling test. Keep format and audience mostly constant; change one variable such as hook, first frame, CTA, or visual system.

### High Confidence

Use high confidence when:

- a pattern wins across multiple posts, formats, or time windows
- the same audience action repeats, such as saves, shares, follows, or link taps
- results remain strong beyond the first 24 hours

Action: turn the pattern into a reusable template, update `INSTAGRAM_MEMORY.md`, and plan 2-3 related tests.

## Choosing The Next Post

Use this order:

1. If a recent post crushed the target metric, create a controlled sibling of that post.
2. If metrics are low or unclear, choose the current recommended draft.
3. If the account needs discovery, prioritize reach-first identity/trust or clear beginner education.
4. If saves/shares are the goal, prioritize educational carousel templates.
5. If comments are the goal, use a community prompt template.
6. If trust or profile visits are the goal, use privacy/trust or charting-boundary education.
7. If traffic is the goal, use a Story/link-sticker flow or product-light post with a natural link-in-bio CTA.

## External Research Rule

Every research-informed post should cite a source pattern, not a copied source.

Good source pattern:

- beginner self-identification
- question-led education
- one idea per carousel slide
- soft community prompt
- routine-based product demo
- privacy proof with user benefit

Reject source patterns that depend on:

- fear, shame, or urgency
- medical authority Well Within does not have
- pregnancy outcome claims
- prediction/status/effectiveness framing
- giveaways without a compliance plan
- generic decorative cards or copied layouts

## Creative Variable Rules

Change one primary variable when possible:

- concept
- audience angle
- format
- hook
- first frame
- visual style
- caption length
- CTA
- posting time

If changing more than one variable, mark the post as exploratory in `INSTAGRAM_EXPERIMENT_LOG.md`.

## When To Use A Stronger Creative Review

Use the strongest available model or a creative-direction review step when:

- the visual system is new
- external references are being interpreted
- image generation is involved
- the post is medical-adjacent
- the final asset feels "good enough" but not clearly on-brand
- the agent is unsure whether the creative is copied, generic, fake, or too ad-like

Lower-capability agents should choose a template and deterministic layout instead of improvising.
