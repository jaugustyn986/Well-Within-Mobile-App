# Template: Product-Light Demo

## Content Job

Product-light app utility.

## Best Used When

- The goal is profile visits, link taps, or app understanding.
- The account has already posted recent reach/trust content.
- The demo can use real app UI without exposing private data.

## Target Metric

Primary: profile visits and link taps.  
Secondary: follows and saves.

## Source Patterns

- Routine-based product demos from fertility and wellness apps.
- Simple "daily habit" workflows.
- Feature proof without overclaiming.

## Format Options

- Reel screen recording.
- Carousel with real app UI screenshots.
- Story with link sticker for direct App Store traffic.

## Structure

1. Hook with the routine or problem.
2. Show the real app UI.
3. Highlight one utility, such as daily entry, chart history, or export.
4. Tie utility back to observation-based clarity.
5. Link-in-bio or Story link CTA.

## Visual System

Use an app-led visual only because the post is proving product utility. Prefer a real phone-on-desk or charting routine context around the UI when possible.

Use real app UI only. Keep app colors, warm neutrals, soft spacing, calm hierarchy, and restrained branding. Do not generate fake screenshots.

## Copy Patterns

- "A calmer daily charting habit starts with one entry."
- "Record the observation. Let the pattern build over time."
- "A charting app should help you notice, not pressure you to predict."

## Caption Pattern

Hook -> product utility -> link-in-bio CTA.

## Safety Notes

- Do not show private health data.
- Do not claim prediction, diagnosis, pregnancy outcomes, or contraception effectiveness.
- Do not use generated or approximate app UI.
- If using the app icon, composite the real `apps/mobile/assets/icon-1024.png`.
